Deficiências visuais

A deficiência visual é a perda ou a redução da capacidade visual em ambos os olhos em caráter definitivo. Pessoas com algum tipo de deficiência do tipo visual, tem dificuldades ao acessar websites se, não implementados de uma forma acessível.

Temos diversas áreas em um site que precisam ser melhoradas para obter a acessibilidade WEB para deficientes visuais. Separamos em tópicos (de acordo com a W3C) para ser mais fácil para você, desenvolvedor, conseguir tornar sua web mais inclusiva! Segue o fio abaixo:

CORES:

Os cuidados com cores são importantíssimos no desenvolvimento de páginas web. O WCAG 2.0 (Web Content Accessibility Guidelines) é bem direto com relação a isso: A cor não deve ser utilizada como o único meio visual de transmitir informações. Isso significa que você não deve utilizar indicações como “clique no botão verde para continuar” ou, “campos marcados em vermelho são obrigatórios”. Por isso mesmo, o E-Mag, modelo de acessibilidade do Governo Eletrônico diz que o mínimo de contraste entre o plano de fundo e o primeiro plano deve ser de 3 para 1. Se você tem dúvidas, na última página do E-Mag existe uma tabela de cores que faz o contraste adequado com preto e branco (já em hexadecimal). No WCAG 2.0, esse valor é de 4,5 para 1.

  • Contraste mínimo de 4:5:1
  • Contraste em diversos fundos e não apenas no branco
  • Usar legendas diretamente sobre a imagem, gráficos e símbolos ou texturas, linhas e traços diversos.
  • Manter todos os links com a mesma cor e seguir sempre um mesmo padrão para os títulos e elementos textuais.
  • Distinção entre verde e vermelho é bem difícil para um daltônico
  • Utilize ícones, texto e até mesmo textura para deixar uma informação clara.
  • Mais brilho, mais saturação, mais matiz, mais tudo!

CONTEÚDO DO SITE:
  • Prover alternativas, tais como transcrições em texto acessível para conteúdo de áudio, como os podcasts;
  • Prover recursos como audiodescrição e legendagem para arquivos de vídeo.
  • Estrutura: o HTML estrutura a informação, dando significado (semântica) aos elementos presentes;
  • Aparência: cores, tamanho, diagramação e posicionamento dos elementos são dados pelas folhas de estilo (CSS);
  • Comportamento: alguns elementos do HTML são interativos, por exemplo o link. Mas a interação prevista para além do que está contido nos elementos HTML, chamada comportamento, pode ser obtida por meio de linguagem de programação, como o Javascript.
  • Boa organização estrutural e semântica do HTML
  • Não inclua conteúdo que pisca em taxas e padrões específicos
  • Fornecer mecanismos para desligar as animações, a menos que sejam essenciais
  • Ferramenta de lupa e zoom
